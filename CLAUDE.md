## Commands

- `npm run dev` — Vite dev server (HMR). Also regenerates `src/routeTree.gen.ts`.
- `npm run build` — Production build targeting Cloudflare Workers.
- `npm run build:dev` — Production build with `--mode development` (keeps dev env vars / unminified output for debugging deploys).
- `npm run preview` — Preview the production build.
- `npm run lint` — ESLint over the repo.
- `npm run format` — Prettier write.

No test framework is configured; do not invent a test command. No CI config.

## Architecture

TanStack Start v1 (SSR) on Cloudflare Workers, React 19, file-based routing, Tailwind 4 + shadcn/ui (new-york style, Radix primitives). TypeScript with `@/*` → `./src/*`. shadcn CLI additions land in `src/components/ui/` per `components.json`; hand-written app components live one level up in `src/components/`.

Request flow:
1. **`src/server.ts`** — CF Worker entry. Wraps TanStack Start's SSR handler with `normalizeCatastrophicSsrResponse`, which recovers real errors h3 swallowed into generic `{"unhandled":true,"message":"HTTPError"}` 500s by reading captures from `src/lib/error-capture.ts` and returning a branded static HTML error page (`src/lib/error-page.ts`).
2. **`src/start.ts`** — exports `startInstance` with error middleware.
3. **`src/router.tsx`** — `getRouter()` builds the router and QueryClient.
4. **`src/routes/__root.tsx`** — root shell (dark theme, `<Outlet/>`, QueryClientProvider, 404/error UIs). Pages use `<Layout>` (Navbar + Footer + Chatbot).

## Critical config rules (easy to break)

- **`vite.config.ts` wires plugins explicitly**: `tsConfigPaths`, `tailwindcss`, `tanstackStart`, `viteReact`, `cloudflare` — in that order. `tanstackStart` must come before `viteReact`; `cloudflare` must be last. The TanStack Router plugin is auto-registered by `tanstackStart`, so do not add `@tanstack/router-plugin` separately.
- `tanstackStart.server.entry: "server"` redirects the bundled SSR entry to `src/server.ts`. `wrangler.jsonc` `main` alone is insufficient — `@cloudflare/vite-plugin` builds from the Vite config.
- **`src/routeTree.gen.ts` is generated by the TanStack Router plugin — never edit it.** It's in `.prettierignore`. Regenerate via `npm run dev` or `npm run build`.
- ESLint `no-restricted-imports` blocks the `server-only` package (Next.js pattern). Use `*.server.ts` filenames or `@tanstack/react-start/server-only`.
- `bunfig.toml` enforces a 24h minimum-release-age supply-chain guard.
- Prettier: 100 print width, trailing commas everywhere, double quotes.
