# AGENTS.md

## Stack

- **TanStack Start v1** (React 19, file-based router, SSR on Cloudflare Workers)
- **TanStack Router v1** + **TanStack Query v5** — router created in `src/router.tsx`
- **Tailwind CSS 4** (via `@tailwindcss/vite`) + **shadcn/ui** (new-york style, Radix primitives)
- **framer-motion**, **Vite 7**, **Wrangler** (Cloudflare deploy)
- **TypeScript** with `@/*` → `./src/*` path alias

## Key commands

| Command | Action |
|---|---|
| `npm run dev` | Vite dev server (HMR) |
| `npm run build` | Production build (CF Workers) |
| `npm run preview` | Preview production build |
| `npm run lint` | ESLint (`eslint .`) |
| `npm run format` | Prettier (`prettier --write .`) |

No test framework is configured. No CI config.

## Entrypoints

- **CF Worker entry**: `src/server.ts` — wraps TanStack Start SSR entry with error recovery (h3-swallowed errors, unhandled rejections)
- **TanStack Start instance**: `src/start.ts` — exports `startInstance` with error middleware
- **Router**: `src/router.tsx` — `getRouter()` creates router + QueryClient

## Routes

All in `src/routes/`:
- `__root.tsx` — root shell (dark theme, Outlet + QueryClientProvider, 404/error components)
- `index.tsx` — homepage (Hero, Solutions, CTA)
- `tools.tsx` — `/tools` (filterable grid)
- `blog.tsx` — `/blog` (featured + tabbed list)
- `contact.tsx` — `/contact` (info sidebar + form)

Every page uses `<Layout>` component (Navbar + Footer + Chatbot).

## Config rules

- `vite.config.ts` wires `tanstackStart`, `@vitejs/plugin-react`, `@tailwindcss/vite`, `vite-tsconfig-paths`, and `@cloudflare/vite-plugin` directly. Order matters: `tsConfigPaths` and `tailwindcss` before `tanstackStart`; `cloudflare` last.
- Server entry redirect: `tanstackStart: { server: { entry: "server" } }` — points to `src/server.ts`
- ESLint: `no-restricted-imports` blocks `"server-only"` package (no Next.js pattern). Use `*.server.ts` naming or `@tanstack/react-start/server-only`.
- `bunfig.toml`: 24h minimum-release-age supply-chain guard.
- Prettier: 100 print width, trailing commas everywhere, single quotes off.

## SSR error recovery (`src/server.ts`)

When h3 swallows a handler throw into a `{"unhandled":true,"message":"HTTPError"}` 500 JSON response, `normalizeCatastrophicSsrResponse` recovers the real error from `globalThis.addEventListener("error")` capture (`src/lib/error-capture.ts`) and returns a branded static HTML error page instead.
